# author_corpus

This directory holds reference texts written by the author. They are used by the `/extract-author-style` skill to generate a stylometric fingerprint of the author's voice.

The generated profile (`style_profile.md`) is then used by compatible skills via the `--preserve-voice` flag to review texts while preserving — not erasing — the author's distinctive voice.

## Structure

```
author_corpus/
├── README.md               # this file
├── style_profile.md        # generated by /extract-author-style (committed)
├── *.pdf                   # academic papers, thesis (gitignored — add your own)
└── my-medium-KB/
    └── posts/              # exported Medium/blog posts as HTML (gitignored — add your own)
```

## Setup

1. Add your reference texts to this directory:
   - Blog posts (HTML) → `my-medium-KB/posts/`
   - Academic papers, thesis (PDF) → root of `author_corpus/`

2. Run the skill to generate your style profile:
   ```
   /extract-author-style
   ```

3. Review and edit `style_profile.md` as needed — it's a plain markdown file.

## Usage

Once the profile exists, activate voice-aware review with:

```
/style-clarity --preserve-voice
```

To regenerate the profile from scratch, run `/extract-author-style` again.

> **Note:** Personal files (PDFs, HTML exports) are gitignored. Only `style_profile.md` and this README are tracked.
